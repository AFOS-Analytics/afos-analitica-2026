# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AFOS Analytics, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, send an email to: **security@afos-analytics.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: within 48 hours
- **Initial assessment**: within 5 business days
- **Fix and disclosure**: coordinated with reporter

## Scope

This policy applies to:
- The AFOS Analytics codebase ([github.com/AFOS-Analytics/afos-analitica-2026](https://github.com/AFOS-Analytics/afos-analitica-2026))
- The production site ([afos-analytics.com](https://afos-analytics.com))
- Associated APIs and infrastructure

## Out of Scope

- Third-party services (Polymarket, Kalshi, Google News, Vercel, Neon, Upstash)
- Social engineering attacks
- Denial of service attacks

## Recognition

We appreciate responsible disclosure and will credit reporters (with permission) in our release notes.
